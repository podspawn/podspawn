     Phase 5a: Local Mode Without SSH

     Context

     Local mode currently hooks into macOS sshd (AuthorizedKeysCommand), which
     conflicts with OrbStack on port 22 and requires root, key registration, and
      sshd configuration. This pivot removes SSH from local mode entirely.
     Containers are managed via podspawn create/run/shell/stop using Docker
     exec. Server mode (sshd hook for remote access) stays unchanged.

     Approach

     Reuse the existing spawn.Session and state DB with minimal changes. Map
     machine names to the existing (user, project) composite key вҖ” user = OS
     user, project = machine name. Add LocalDefaults() for user-local paths
     (~/.podspawn/state.db). New commands create and run build sessions
     identically to spawn/shell but with local config.

     Implementation Order

     1. Config: Add LocalDefaults() and LocalConfig

     Files: internal/config/config.go, internal/config/client.go

     config.go вҖ” Add LocalDefaults() *Config:
     - Same defaults as Defaults() but with user-local paths:
       - DBPath: ~/.podspawn/state.db
       - LockDir: ~/.podspawn/locks/
       - MaxLifetime: 24h (longer for local dev)
     - Reuse existing Defaults() and override the paths

     client.go вҖ” Add LocalConfig struct to ClientConfig:
     type LocalConfig struct {
         Image       string  `yaml:"image"`
         Shell       string  `yaml:"shell"`
         CPUs        float64 `yaml:"cpus"`
         Memory      string  `yaml:"memory"`
         GracePeriod string  `yaml:"grace_period"`
         MaxLifetime string  `yaml:"max_lifetime"`
         Mode        string  `yaml:"mode"`
     }
     type ClientConfig struct {
         Servers ServerRouting `yaml:"servers"`
         Local   LocalConfig   `yaml:"local"`
     }

     Tests: internal/config/config_test.go вҖ” verify LocalDefaults paths, verify
     client config local section parses and overrides.

     2. Root config fallback for local-mode commands

     File: cmd/root.go

     Add create, run, shell to configOptionalCommands. Add a localModeCommands
     map:
     var localModeCommands = map[string]bool{
         "create": true, "run": true, "shell": true,
     }

     In PersistentPreRunE fallback, when server config missing and command is in
      localModeCommands, use config.LocalDefaults() instead of
     config.Defaults(). Also attempt to load ~/.podspawn/config.yaml local
     section and apply overrides.

     3. Session.Ensure() method

     File: internal/spawn/spawn.go

     Add method:
     func (s *Session) Ensure(ctx context.Context) (string, error) {
         if s.Store == nil {
             return "", fmt.Errorf("state store required for machine creation")
         }
         containerName, isNew, err := s.ensureContainerWithState(ctx)
         if err != nil {
             return "", err
         }
         s.ensurePodfileParsed()
         s.runHooks(ctx, containerName, isNew)
         return containerName, nil
     }

     Calls ensureContainerWithState() (which creates container + records in DB
     with connections=1) and returns without attaching or disconnecting. The
     connections=1 acts as "machine is alive" marker.

     Tests: internal/spawn/spawn_test.go вҖ” test Ensure creates container,
     records session, doesn't exec.

     4. Shared local session builder

     File: cmd/local.go (new)

     Helper buildLocalSession(name string) (*spawn.Session, *state.Store, 
     error):
     1. Get OS username from os.Getenv("USER")
     2. Connect Docker runtime
     3. Parse config values (memory, grace period, max lifetime) from global cfg
     4. Open state DB at cfg.State.DBPath
     5. Build spawn.Session with Username=osUser, ProjectName=name
     6. Return session + store (caller defers store.Close)

     This deduplicates session-building across create, run, and the updated
     shell.

     5. podspawn create <name> command

     File: cmd/create.go (new)

     Usage: podspawn create <name> [--image IMAGE]

     Implementation:
     1. buildLocalSession(name)
     2. Override image from --image flag if provided
     3. Call sess.Ensure(ctx)
     4. Print: created machine "{name}"
     5. Close store

     Start with --image only. Defer --podfile, --project, --branch, --with,
     --hostname to Phase 5b.

     Tests: cmd/create_test.go вҖ” test argument parsing, verify session built
     correctly.

     6. podspawn run <name> command

     File: cmd/run.go (new)

     Usage: podspawn run <name> [--image IMAGE]

     Implementation:
     1. buildLocalSession(name)
     2. Override image from --image flag if provided
     3. Set sess.Mode = "destroy-on-disconnect", sess.GracePeriod = 0 (ephemeral
      default)
     4. Clear SSH_ORIGINAL_COMMAND
     5. Call sess.RunAndCleanup(ctx) вҖ” creates, attaches shell, destroys on exit
     6. os.Exit(exitCode)

     Tests: cmd/run_test.go вҖ” test ephemeral mode settings applied.

     7. Update podspawn shell for local mode

     File: cmd/shell.go

     Replace the session-building code with buildLocalSession(name) from
     local.go. Already in configOptionalCommands via step 2. The existing
     parseShellTarget returns ("", name) for bare names, then fills user from
     $USER. Works as-is with the local session builder.

     8. Update podspawn stop for local mode

     File: cmd/stop.go

     Detect local mode (no server config loaded вҶ’ localModeCommands fallback was
      used). In local mode, interpret bare arg as machine name: (os_user, arg).
     In server mode, keep current behavior: user[@project].

     9. Update podspawn list for local mode

     File: cmd/list.go

     In local mode, rename PROJECT column to NAME in tabwriter output. Drop USER
      column (it's always the current user). Add IMAGE column. Detection: check
     if command used local config fallback.

     10. Rewrite up.sh local mode

     File: scripts/up.sh

     Replace lines 371-456 (local mode section). New flow:
     1. Check Docker accessible (not sshd)
     2. Add user to docker group if needed
     3. Create ~/.podspawn/config.yaml with local defaults if missing
     4. Print usage: podspawn run mydev, podspawn create mydev, etc.

     Remove OrbStack detection section entirely (lines 207-248) вҖ” not needed
     when local mode doesn't touch port 22.

     Remove macOS sshd prerequisites (host keys, Remote Login) from local mode
     path вҖ” only keep for server mode.

     11. Nuke OrbStack-specific code

     Files: scripts/up.sh, internal/doctor/doctor.go

     - Remove ORBSTACK_MODE variable and all branches
     - Remove checkPort22() from doctor (or keep for server-mode doctor only)
     - Remove the OrbStack choice menu from up.sh
     - Remove the shell-mode local path from up.sh (replaced by step 10)

  Problem: User on container creation requires the user in /etc/passwd. Bare

вҸә Design: Non-Root Container Users

  Pattern: Every tool (VS Code Dev Containers, Codespaces) creates the user once
   at container start, then execs as that user. Nobody creates users at exec
  time.

  Approach:

  1. Container starts as root with sleep infinity (unchanged)
  2. Immediately after start, exec a user-creation script as root (one time,
  idempotent)
  3. All subsequent execs set User: username on ExecOptions
  4. UID 1000:1000 standard (handle ubuntu's existing ubuntu user at 1000)
  5. Sudo installed via apk add shadow sudo on Alpine, sudoers.d for
  passwordless sudo
  6. dotfiles.go clones to /home/<user>/dotfiles not /root/dotfiles

  User creation script (runs as root once):
  id -u USERNAME 2>/dev/null || {
    # Alpine: install shadow + sudo for useradd/sudo support
    command -v apk >/dev/null && apk add --no-cache shadow sudo >/dev/null 2>&1
    # Handle UID 1000 conflict (ubuntu:24.04 has 'ubuntu' at 1000)
    EXISTING=$(getent passwd 1000 2>/dev/null | cut -d: -f1)
    [ -n "$EXISTING" ] && [ "$EXISTING" != "USERNAME" ] && usermod -u 65534
  "$EXISTING"
    groupadd --gid 1000 USERNAME 2>/dev/null || true
    useradd --uid 1000 --gid 1000 -m -s SHELL USERNAME
    echo 'USERNAME ALL=(root) NOPASSWD:ALL' > /etc/sudoers.d/USERNAME
    chmod 0440 /etc/sudoers.d/USERNAME
  }

  - 2 SSHD tests (apt-get install) вҖ” user has sudo, so sudo apt-get works
  - dotfiles.go вҖ” change /root/dotfiles to /home/<user>/dotfiles
  - 16 other SSHD tests вҖ” unaffected (write to /tmp, check stdout/exit codes)
